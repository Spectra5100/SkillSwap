# Test protocol — SkillSwap

Les testeurs ont été invités à utiliser le prototype SkillSwap comme s’il s’agissait d’une application réelle.  
Ils devaient explorer l’interface et réaliser plusieurs actions clés afin d’évaluer la compréhension du concept, la clarté du parcours et la facilité de navigation.

Les tâches demandées étaient les suivantes :
- parcourir la page d’accueil et découvrir des compétences,
- sélectionner une compétence et consulter ses détails,
- tenter d’accéder au contenu associé (lecture, PDF, Zoom),
- naviguer entre les pages (chat, profil, retour),
- identifier comment proposer une nouvelle compétence.

Les tests ont été menés de manière **modérée**, avec la présence de deux observateurs qui :
- demandaient au testeur de verbaliser ce qu’il faisait et ce qu’il pensait,
- n’intervenaient pas dans les choix de navigation,
- n’apportaient d’aide qu’en cas de blocage.

Les réactions, hésitations, commentaires et difficultés rencontrées ont été notés afin d’identifier les points de friction et les axes d’amélioration du prototype avant le MVP.
